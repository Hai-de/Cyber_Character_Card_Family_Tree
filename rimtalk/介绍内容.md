# rimtalk是什么？

rimtalk是一个rimworld（边缘世界）的mod，，他可以让小人使用ai生成的内容来对话让你的小人看起来栩栩如生？（看起来罢了）

# 前言

当我回顾我整个玩ai和酒馆的历程发现酒馆和ai在我眼中已经没有了当初的神秘迷雾笼罩，因此距离不再能产生美，这期间我已经很少打开酒馆，大部分时间都是在瓜区水聊天，哪怕是前一段时间我打开酒馆游玩发现gemini-3pro的八股文依旧没有任何改观。
近期talk更新了所谓的高级预设功能，我认为是时候该写一个talk的预设玩玩了，不然我非常无聊。

# 我可能面临的问题？

> 这个和酒馆的预设有不同？

- talk的预设可以通过{{}}来内嵌代码判断和变量管理，这个设计我曾经在risu上体验过相同的感受，并且我发现talk似乎可以直接调用/获取rim本身的函数/对象/类？（对c#代码我并不了解）我认为这比酒馆的睿智{{}}要有趣
- talk由于是作为辅助rim游戏本身的mod，这和酒馆以ai返回内容为主体不一样，这意味着talk不适合使用带有思维链的模型，对于ai的响应速度有要求，也可以认为不能给ai太重的任务
- talk的ai不需要像酒馆的ai一样需要描绘环境内容，因为环境信息都是talk将特定游戏数据发送给ai来使用
- 根据我的观察，贴吧那些分享talk的人对于ai生成的所谓的角色扮演通常格式是“小人名字（括号内是角色的动作和神态描写）括号外面是角色所说的话语”，小人名字是返回的json结构中必须要有的键，这种交互形式在我看来和国内那种所谓的ai聊天app差不多，或者说相当原始而有效？
- if语句能有效分割提示词，减少ai的注意力消耗

> 要如何处理多mod导致的提供给ai的提示词可能存在的冲突问题？

- 疑似没有办法，我估计只能从提示词处缓解提供给ai的提示词内部冲突？
